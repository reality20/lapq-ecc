# 01 — Mathematical Foundations

## 1.1 The secp256k1 Curve

The secp256k1 curve is defined over the prime field F_p where:

```
p = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEFFFFFC2F
  = 2²⁵⁶ − 2³² − 2⁹ − 2⁸ − 2⁷ − 2⁶ − 2⁴ − 1
  = 2²⁵⁶ − 4294968273
```

The curve equation is:

```
E: y² = x³ + 7   (over F_p)
```

That is, a = 0 and b = 7 (short Weierstrass form with a = 0).

### Generator Point

```
G_x = 0x79BE667EF9DCBBAC55A06295CE870B07029BFCDB2DCE28D959F2815B16F81798
G_y = 0x483ADA7726A3C4655DA4FBFC0E1108A8FD17B448A68554199C47D08FFB10D4B8
```

### Group Order

```
n = 0xFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFEBAAEDCE6AF48A03BBFD25E8CD0364141
```

The group E(F_p) is cyclic of order n (a 256-bit prime), with cofactor h = 1.

## 1.2 The Elliptic Curve Discrete Logarithm Problem (ECDLP)

**Problem statement:** Given two points P, Q ∈ E(F_p) where Q = kP for some unknown integer k ∈ [0, n−1], find k.

In the context of secp256k1:
- P = G (the standard generator)
- Q is a public key
- k is the private key (secret scalar)

The hardness of ECDLP is the security foundation for ECDSA signatures used in Bitcoin, Ethereum, and many other systems.

**Classical hardness:** The best known classical algorithms (Pollard's rho) require O(√n) ≈ 2¹²⁸ group operations — computationally infeasible.

**Quantum complexity:** Shor's algorithm solves ECDLP in polynomial time: O(log³ n) quantum gates.

## 1.3 Shor's Algorithm for ECDLP

Shor's algorithm for ECDLP differs from the integer factoring variant. The ECDLP version uses the **abelian hidden subgroup structure** of the elliptic curve group.

### Problem Reformulation

Given P, Q = kP, define the function:

```
f(a, b) = aP + bQ = (a + bk)P
```

This function is periodic: f(a, b) = f(a', b') if and only if:

```
a + bk ≡ a' + b'k  (mod n)
```

The hidden subgroup (kernel) is generated by (k, −1) in Z_n × Z_n.

### Algorithm Steps

1. **State Preparation:** Initialize two quantum registers |a⟩ and |b⟩, each of ⌈log₂ n⌉ = 256 qubits, in uniform superposition.

2. **Oracle Evaluation:** Compute f(a, b) = aP + bQ into an output register:
   ```
   |a⟩|b⟩|O⟩  →  |a⟩|b⟩|aP + bQ⟩
   ```

3. **Quantum Fourier Transform:** Apply QFT to the two input registers |a⟩ and |b⟩.

4. **Measurement:** Measure both registers, obtaining values (c, d).

5. **Classical Post-Processing:** From (c, d), recover k using:
   ```
   ck + d ≡ 0  (mod n)   ⟹   k ≡ −d · c⁻¹  (mod n)
   ```
   (with high probability, subject to the measurement yielding a useful result).

### Success Probability

A single run yields a useful (c, d) with probability ≥ 1/poly(log n). After O(1) repetitions, the correct k is recovered with overwhelming probability.

### The Core Quantum Operation

The bottleneck is step 2: computing the **controlled elliptic curve scalar multiplication**. This requires:

- Modular arithmetic (addition, multiplication, inversion) in F_p
- Elliptic curve point addition and doubling
- All operations must be **reversible** (unitary)
- The scalars a, b are in superposition — they control the point operations

## 1.4 Modular Arithmetic Foundations

All arithmetic operates over F_p where p = 2²⁵⁶ − 4294968273.

### Special Form of p

The Mersenne-like structure p = 2²⁵⁶ − c where c = 4294968273 < 2³³ makes modular reduction particularly efficient:

```
x mod p:
  If x = x_hi · 2²⁵⁶ + x_lo, then:
  x mod p ≡ x_lo + c · x_hi  (mod p)
```

This is because 2²⁵⁶ ≡ c (mod p). The reduction requires at most a few iterations since c is small.

### Required Operations

| Operation | Classical Cost | Quantum Cost (reversible) |
|---|---|---|
| Modular addition | O(n) | O(n) gates, O(1) ancillae |
| Modular multiplication | O(n²) | O(n²) gates, O(n) ancillae |
| Modular inversion | O(n²) (extended GCD) | O(n²) gates, O(n) ancillae (or avoided) |
| Modular squaring | O(n²) | O(n²) gates, O(n) ancillae |

where n = 256 is the bit-length of the field elements.

## 1.5 Elliptic Curve Group Law

For P₁ = (x₁, y₁), P₂ = (x₂, y₂) on E: y² = x³ + 7:

### Point Addition (P₁ ≠ ±P₂)

```
λ = (y₂ − y₁) / (x₂ − x₁)
x₃ = λ² − x₁ − x₂
y₃ = λ(x₁ − x₃) − y₁
```

### Point Doubling (P₁ = P₂)

```
λ = (3x₁²) / (2y₁)       [since a = 0, simplified]
x₃ = λ² − 2x₁
y₃ = λ(x₁ − x₃) − y₁
```

### Identity Element

The point at infinity O serves as the identity: P + O = O + P = P.

### Projective Coordinates

To avoid field inversions in the quantum circuit (which are expensive and hard to make reversible), we use **Jacobian projective coordinates**:

A point (X : Y : Z) represents the affine point (X/Z², Y/Z³).

Point addition and doubling in Jacobian coordinates use only multiplication and addition — no division. See [Chapter 4](04_ec_point_operations.md) for the explicit formulas.
